搭建ghost时遇到的坑

2017-08-05

1.https协议照成的重定向循环

搭建ghost的时候遇到一个问题就是因为加了锁的https协议照成的重定向循环
需要在

这是一个http的头部，来说明是用什么协议进行通信

2. 没有config.example.js的文件

在下载ghost是从网上查了下发现大部分教程都是下载ghost文件解压后有一个config.example.js的文件，在这里设置vps的url连接，从官网下载最新的zip结果发现没有这个文件，一开始以为自己下错了，后来才发现是软件升级了，大版本更新，由0.7升级到了1.0安装的方式变了。后来从ghostchina下载的中文版的0.7版本装的。

3.需要安装什么

安装中发现不需要安装php与mysql，只需要nginx与nodejs及课运行。

4.nodejs的版本不对

在安装中进行nodejs安装，发现版本不对，需要Node 0.10.x 版本，同时支持 Node 0.12.x 和 4.2.x 版本，那怎么办？发现可以用nvm或者n进行nodejs的版本管理，经过两个比较，发现nvm是全局的改变nodejs版本而n有一定限制，nvm比较好。nvm+版本号，yum remove进行卸载

5.使用npm进行相应nodejs的安装

npm install –production，直接进行安装

6.程序无法再关闭连接后持续后台运行

我们需要进行守护程序来让ghost在后台持续运行，我们使用进程守护程序“Forever”，通过npm install forever -g进行安装

7.使用NODE_ENV变量与forever命令

NODE_ENV=production forever start index.js，来让ghost启动，前面的NODE_ENV变量表示以什么环境进行运行，这个是生产模式，还有开发模式，可以选择 forever stop index.js 关闭 NODE_ENV=production forever restart index.js 重启 forever list 查看运行状态
